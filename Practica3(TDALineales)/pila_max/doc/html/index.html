Pila Max (TDAs Lineales)

Este pequeño proyecto recopila varias implementaciones del tipo de dato Pila con Máximo. Con este tipo de dato se obtiene una pila de enteros en la que se guarda no solo el entero introducido sino el entero que en el momento de la insercción es el mayor de la Pila. Lo interesante del proyecto es que la implementación de lo que hay por debajo de este TDA se ha realizado usando varios tipos de estructuras: una lista (LIFO), un vector y una cola (FIFO). En el siguiente fichero puede verse una prueba de uso de cada uno de los tipos.

#include <iostream>
//Debería funcionar usando pila_max p, pero no lo hace.
#include "pila_max.h"
using namespace std;
int main(){
//Prueba de la implementación de pila con máx implementada con una lista.
//pila_max_lista p;
//Prueba de la implementación con un vector dinámico.
//pila_max_vd p;
//Prueba de la implementación con una cola.
pila_max_cola p;
//int i;
p.poner(5);
p.poner(10);
p.poner(3);
p.quitar();
p.poner(12);
p.poner(1);
int i=0;
while (!p.vacia() ){
cout << "paso " << i;
i++;
elemento x = p.tope();
p.quitar();
cout << x << endl;
}
return 0;
}


Además de esto se ha implementado el tipo de dato Cola con Máximo que viene a tener el mismo principio que la Pila anterior solo que esta vez la implementación se ha realizado con una Pila. Es decir, se ha tenido que realizar la implementación para que Lista funcione desde fuera como una Cola. Puede verse un ejemplo de ejecución en el siguiente fichero:

#include <iostream>
#include <cola_max.h>
using namespace std;
//Programa de ejemplo del T.D.A. Cola con máximo, que funciona de forma similar a pila con máximo pero esta vez se trata de
//una cola con máximo y debajo tiene implementada una pila.
int main(void){
//Declaramos una cola
cola_max cola;
//Introducimos en la cola los valores 5,10,3,12 y 1.
cola.poner(5);
cola.poner(10);
cola.poner(3);
cola.poner(12);
cola.poner(1);
while (!cola.vacia() ){
elemento x = cola.inicio();
cola.quitar();
cout << x << " ";
}
cout << endl;
//Al ir extrayéndolos de la cola se debería ver por terminal la secuencia:
//[5 | 5] [10 | 10] [3 | 10] [12 | 12] [1 | 12]
}
